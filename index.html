Data Last

Data Last means "set up all your functions before any data comes in". This is normally inconvenient because most of the functions we'd like to set up live on the prototypes of the data, e.g. Array.prototype.map. rubico takes these methods off of the prototype so you can put the data last.

Function Composition

Since data is last, it's easy to compose functions; just pipe them together. If you need to act on a collection, e.g. an array, reach for map, filter, or reduce. In this example, we compose squaredOdds from square and isOdd.

By running the example as is, we expect the input array [1, 2, 3, 4, 5] to transform into the output array [1, 9, 25].

Optional Asynchrony

You can pass rubico sync or async functions. When you pass async functions, rubico handles the promise resolution for you. When you pass sync functions, rubico evaluates everything synchronously. Let's take a look with map and filter.
