[a]synchrony

You can pass rubico synchronous or asynchronous functions. When any functions are asynchronous, rubico handles the promise resolution for you. That means you can stop worrying about explicitly calling .then or Promise.all.

We can concisely make five requests to an API in parallel.

Data Last

Data Last means "data is supplied to functions as the last step". This is desirable because it makes function composition a breeze. However, it is inconvenient in practice because most of the functions you'd want to set up reside on the prototypes of the data, e.g. Array.prototype.map. rubico takes these methods off of the prototype so you can put the data last.

You should see two identical arrays here.

Function Composition

Since data is last, it's easy to compose functions; just pipe them together. If you need to act on a collection, e.g. an array, use map, filter, or reduce. In this example, we compose squaredOdds from square and isOdd.

By running the example as is, we expect the input array [1, 2, 3, 4, 5] to transform into the output array [1, 9, 25].

Shape Control

There will be times when you'll want to control the shape of your data. For times like these, use fork or assign. Use get in conjunction with these to manipulate payloads freely.
